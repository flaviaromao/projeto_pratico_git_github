git init (para inicializar o git)
git status (para verificar o status do repositório e dos arquivos alocados nele)
git add "nome_arquivo.txt" (exemplo de como adicionar um arquivo no controle de versões)
git add . (para adicionar todos os arquivos do repositório no controle de versão)
git commit -m "Descreve-se sobre a versão (o que mudou, do que se trata, etc...)"
git remote add origin url_do_repositorio (para definir para qual repositório do GitHub os arquivos serão enviados com o comando push)
git push (para enviar os arquivos para o repositório no GitHub)
git reflog (para verificar o log de versões)
git reset --hard id_da_versão (para voltar para determinada versão em caso de erro)

git branch (para verificar as branchs disponíveis)
git branch nome_branch (para criar uma nova branch)
git checkout nome_branch (para alterar de branch)
git merge nome_branch (para trazer um arquivo de uma branch para outra. Por exemplo, nome_branch seria o nome da branch de dev, e devemos rodar este código na branch principal)

git pull (para atualizar na máquina os arquivos mais recentes que estão na nuvem)

git checkout -b nome_nova_branch nome_branch_base (para criar uma branch utilizando como base uma existente)
